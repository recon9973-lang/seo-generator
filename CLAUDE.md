# 프로젝트 지침

## 🔄 세션 시작·인계 시스템 (토큰 절약 · MUST)
1. **세션 시작 시 먼저 읽어라**: 루트의 **`RESUME.md`**(직전 세션에서 이어갈 작업 — 있으면) → **`PROJECT_STATE.md`**(자동 생성 현황). 이 둘로 재탐색 없이 즉시 오리엔테이션(grep/read 남발 금지 = 토큰 절약).
2. **컨텍스트 관리**: 세션이 길어지면 **`/compact`**(연속성 유지·압축)를 사용자에게 권한다. 질문/오더 **20건 도달** 시 **`checkpoint` 스킬**을 실행(문서화→TODO→`RESUME.md` 생성→커밋)한 뒤 사용자에게 **`/clear`** 입력을 안내한다.
   - **훅(`UserPromptSubmit`)이 10·20건에서 자동 알림**하지만, **웹/원격 환경에선 훅이 안 돌 수 있다**(`/hooks` 미지원). 그 경우 **에이전트가 스스로 세션의 질문/오더 수를 세어** 20건에 도달하면 사용자 지시 없이 **자동으로 checkpoint를 실행**하고(문서화·TODO·RESUME 저장·커밋까지 전부 완료) **그다음 "`/clear` 입력하세요"만 안내**한다.
3. **상태는 반드시 git에** — 컨테이너는 세션마다 초기화된다. `PROJECT_STATE.md`는 자동 생성물이니 손대지 말고 `node scripts/gen-project-state.mjs`(또는 CI)로 갱신.

> 이 블록은 **최강스킬**(연속성·토큰 절약 시스템)로 설치되었다. 정본은 `desktop-tutorial` 저장소.
