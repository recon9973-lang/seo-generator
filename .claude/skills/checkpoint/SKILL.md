# 세션 체크포인트·핸드오프

토큰 누적으로 작업이 어려워지기 전에 **세션을 안전하게 인계**한다. 컨텍스트는 `/clear`로 비우되,
필요한 모든 것을 git에 남겨 **다음 세션이 즉시 이어가게** 한다.

## 언제 실행되나
- **자동**: `UserPromptSubmit` 훅이 이 세션의 질문/오더 수를 센다.
  - **10건(20의 절반)**: 가벼운 **`/compact`** 권장(대화 맥락 유지·컨텍스트만 압축). 인계는 아직 불필요.
  - **20건**: 무거운 **체크포인트 + `/clear`**(아래 절차). 문서화·인계 후 컨텍스트 완전 리셋.
- **수동**: 사용자가 `/checkpoint` 호출.

> **`/compact` vs `/clear`**: 컨텍스트가 길어지지만 계속 이어갈 땐 `/compact`(연속성 유지, 저비용).
> 20건 누적·큰 매듭이 지어졌을 땐 `/checkpoint`→`/clear`(git 인계 후 새 세션). 둘 다 슬래시 명령이라 **사용자가 직접 입력**.

## 실행 절차 (순서대로)

컨트롤 센터 저장소는 **`desktop-tutorial`** 이다. 아래 산출물을 모두 거기에 커밋한다.

### 1. 세션 작업 문서화 (한 곳)
`docs/session-logs/<YYYY-MM-DD>-sNN.md` 파일에 이번 세션에서 처리한 항목을 기록:
- 사용자 요청(오더/질문) 목록 — 20건 요약
- 각 항목의 결과(완료/부분/대기), 관련 커밋 해시·파일
- 발견한 문제·결정사항
> 파일명이 이미 있으면 `-s02`, `-s03`… 로 증가. 기존 로그는 덮어쓰지 않고 append/신규.

### 2. TODO 갱신
`TaskList`로 현재 상태 확인 → 완료분 `completed`, 새 항목 `TaskCreate`, 대기/차단 정리.
문서화한 세션 로그와 TODO가 일치하게 맞춘다.

### 3. 다음 세션 프롬프트 생성 → `RESUME.md` (저장소 루트, git)
다음 세션이 **이 파일만 읽고 즉시 이어가도록** 작성:
```
# RESUME — 다음 세션 이어가기 (<날짜> <시각KST>)
## 지금까지 (핵심만)
- (직전 세션 요약 3~6줄 + 관련 커밋)
## 바로 이어갈 작업
1. (구체적 다음 액션 — 파일·명령·엔드포인트까지)
## 대기/차단
- (사용자 액션 대기 항목: 예 GSC env 입력)
## 주의·제약
- 브랜치/배포 규칙, 비밀키 미노출, 모델ID 미포함 등
## 참고
- 현황은 PROJECT_STATE.md, 지도 위치는 핵심두뇌_MASTER.md
```
과장·날조 금지. 실제 상태만.

### 4. PROJECT_STATE 갱신
`node scripts/gen-project-state.mjs` 실행 → `PROJECT_STATE.md` 최신화.

### 5. 커밋·푸시
위 3개(session-log, RESUME.md, PROJECT_STATE.md)를 `desktop-tutorial` main에 커밋·푸시.
커밋 트레일러 규칙 준수(Co-Authored-By / Claude-Session). 모델 ID 미포함.

### 6. 사용자에게 `/clear` 안내 (중요)
커밋 완료 후, 사용자에게 정확히 이렇게 안내한다:

> ✅ 체크포인트 저장 완료 (session-log · TODO · RESUME.md · PROJECT_STATE.md 커밋).
> **이제 `/clear` 를 입력해 컨텍스트를 비워 주세요.**
> 새 세션이 시작되면 `RESUME.md`를 자동으로 읽어 바로 이어서 작업합니다.

`/clear`는 슬래시 명령이라 **사용자가 직접 입력**해야 한다(하네스가 대신 실행 불가). 반드시 명시적으로 요청할 것.

## 다음 세션에서 (이어받기)
- `SessionStart` 훅 또는 `CLAUDE.md` 지침에 따라 **`RESUME.md`를 먼저 읽는다.**
- RESUME의 "바로 이어갈 작업"부터 재개. 재탐색이 필요하면 `PROJECT_STATE.md`를 본다.
- 이어받은 뒤에는 RESUME이 소비됐음을 인지(다음 체크포인트가 새로 덮어씀).

## 원칙
- 상태는 **반드시 git**에 (컨테이너는 매 세션 초기화됨).
- 문서화는 **정직·간결**. 다음 세션이 헤매지 않을 최소 정보 + 정확한 포인터.
